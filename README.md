# MTProxy Setup

Скрипт установки, обновления и управления [personal_mtproxy](https://github.com/seriyps/personal_mtproxy) с расширенным функционалом.

## Что включено

- Автоматическая установка Erlang последней версии через kerl
- Получение wildcard TLS-сертификата через certbot с автоопределением типа домена
- Панель администратора с мониторингом соединений в реальном времени
- Управление сроком действия прокси — деактивация по дате или вручную
- Страница метрик Prometheus с графиками DC и VM
- Basic Auth защита — пароль хранится на сервере
- Расширенный API (`/api/config`, `/api/proxies`, `/api/connections`, `/api/metrics`)
- Страница-заглушка (парковка домена) на главной странице
- Тёмная и светлая тема интерфейса с запоминанием выбора
- Автообновление скрипта при запуске install/update/reinstall
- Резервные копии с возможностью отката и очистки
- Проверка доступности порта 443 перед установкой

## Требования

- Ubuntu 22.04 или 24.04
- Порт 443 свободен
- Домен с возможностью управления DNS
- Root доступ

## Быстрый старт

```bash
curl -fsSL https://raw.githubusercontent.com/MrAgasferon/mtproxy-setup/main/install.sh -o install.sh
bash install.sh
```

Скрипт задаст несколько вопросов и всё установит автоматически.

## Команды

| Команда | Описание |
|---------|----------|
| `bash install.sh` или `bash install.sh install` | Установка с нуля |
| `bash install.sh reinstall` | Удалить старую установку и установить заново |
| `bash install.sh update` | Обновить до последней версии |
| `bash install.sh backup` | Создать резервную копию |
| `bash install.sh restore` | Восстановить из резервной копии |
| `bash install.sh cleanup` | Управление резервными копиями |
| `bash install.sh status` | Показать состояние сервиса |

## Процесс установки

Скрипт автоматически выполняет следующие шаги:

1. Проверяет доступность порта 443
2. Устанавливает системные зависимости
3. Определяет и собирает последнюю версию Erlang/OTP через kerl
4. Устанавливает rebar3 и certbot
5. Запрашивает домен и получает TLS-сертификат автоматически
6. Клонирует форк personal_mtproxy с патчами
7. Записывает конфигурацию
8. Собирает и устанавливает сервис
9. Настраивает автообновление сертификата через cron

## После установки

Сервис доступен по адресам:

- `https://ВАШ_ДОМЕН/` — страница-заглушка (парковка домена)
- `https://ВАШ_ДОМЕН/admin.html` — панель администратора (требует пароль)
- `https://ВАШ_ДОМЕН/metrics.html` — страница метрик

**Не забудьте открыть порт 443** — скрипт намеренно не трогает firewall:

```bash
# ufw
sudo ufw allow 443/tcp

# iptables
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT
```

## Панель администратора

Открывается по адресу `https://ВАШ_ДОМЕН/admin.html`. Браузер запросит логин и пароль (Basic Auth). Логин — любой, пароль — тот что вы указали при установке.

Возможности панели:
- Создание новых прокси-ссылок с указанием срока действия
- Список всех пользователей с датой регистрации и сроком действия
- Мониторинг активных соединений в реальном времени (обновление каждые 10 сек)
- Копирование ссылок (`t.me`, `tg://`, секрет)
- Переключатель активности — деактивировать без удаления
- Продление срока действия прокси
- Удаление пользователя
- Переключение тёмной/светлой темы

### Срок действия прокси

При создании прокси можно указать дату окончания. Если дата не указана — прокси бессрочный. По истечении срока прокси автоматически деактивируется (проверка каждые 60 секунд). Срок можно продлить через кнопку "Продлить" в панели.

Деактивация прерывает **новые** подключения. Активный сеанс завершится естественным образом. Для немедленного отключения используйте кнопку "Удалить".

## Страница метрик

Открывается по адресу `https://ВАШ_ДОМЕН/metrics.html`. Показывает:
- Пользователей онлайн и активных соединений
- TCP соединения (низкоуровневые ranch)
- Соединения к DC серверам Telegram
- Метрики Erlang VM (память, процессы, GC)
- Автообновление каждые 10 секунд

## Обновление

При обновлении предлагается выбор версии:

```
1) Стабильная (тег v1.0.0) — проверена
2) Последняя (ветка mtproxy-setup-patches)
3) Отмена
```

Стабильная версия — проверенный тег форка. Последняя — актуальная ветка с новыми изменениями. Перед обновлением автоматически создаётся резервная копия.

## Резервные копии

Бэкапы хранятся в `/root/mtproxy_backups/` и **никогда не удаляются автоматически** — ни при `reinstall`, ни при `update`, ни при `restore`.

Каждый бэкап содержит:
- База пользователей (`proxies.dets`)
- Конфигурация (`sys.config`)
- Исходники с патчами
- Бинарники (`/opt/personal_mtproxy`)

Перед каждым `update` автоматически создаётся бэкап. Для удаления старых бэкапов используйте `bash install.sh cleanup`.

## Автообновление скрипта

При запуске команд `install`, `reinstall`, `update` скрипт автоматически проверяет и скачивает свою последнюю версию из GitHub перед выполнением. Если вы всегда запускаете команды через:

```bash
curl -fsSL https://raw.githubusercontent.com/MrAgasferon/mtproxy-setup/main/install.sh -o install.sh && bash install.sh КОМАНДА
```

— вы всегда получаете актуальную версию.

## API эндпоинты

Все эндпоинты защищены Basic Auth.

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/proxies` | Создать новый прокси (параметры: `email`, `expires_at`) |
| `PATCH` | `/api/proxies?subdomain=XXX` | Обновить `active` или `expires_at` |
| `DELETE` | `/api/proxies?subdomain=XXX` | Удалить прокси |
| `GET` | `/api/proxies` | Список всех пользователей |
| `GET` | `/api/config` | Конфигурация прокси (секрет, порт, домен) |
| `GET` | `/api/connections` | Активные соединения по subdomain |
| `GET` | `/api/metrics` | Метрики Prometheus в формате JSON |

## Расположение файлов

| Путь | Содержимое |
|------|------------|
| `/opt/personal_mtproxy/` | Бинарники сервиса |
| `/root/personal_mtproxy/` | Исходники |
| `/var/lib/personal_mtproxy/` | База пользователей и сертификаты |
| `/var/log/personal_mtproxy/` | Логи |
| `/root/mtproxy_backups/` | Резервные копии |
